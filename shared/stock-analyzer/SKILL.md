# 个股深度分析框架

配合 `stock-analysis-base` 和 `industry-analyzer` 使用。本skill专注**个股层面**分析，自下而上验证。

## 分析顺序

```
行业格局 → 公司定位 → 管理层分析 → 财务验证 → 股东博弈 → 产业链验证 → Bull/Bear推演 → 四大师对抗 → 镜子测试 → 结论
```

---

## 〇、分析前必做：信息丰富度评级（A/B/C）

**核心问题**：我对这家公司的信息掌握程度如何？会不会"资料多=确定性高"的幻觉？

在开始分析前，必须先对信息丰富度进行评级：

| 评级 | 特征 | 分析策略 |
|------|------|----------|
| **A级** | 财报完整、研报覆盖多、行业数据充足 | 可以做完整分析，结论置信度高 |
| **B级** | 财报有但研报少，或数据有缺口 | 可以分析，但推算指标需标注置信度 |
| **C级** | 信息稀缺（未上市、新上市、冷门行业） | 只做定性判断，不做定量估值，标注"灰色地带" |

**关键原则**：
- 数据不足时宁可说"不知道"，不用推测伪装确定性
- B/C级信息的分析，所有推算指标必须标注 🟢高/🟡中/🔴低 置信度
- 信息丰富度评级会直接影响最终结论的确定性表述

## 〇、四大师视角对抗机制（借鉴AI Berkshire）

**核心理念**：不是"用巴菲特方法分析一下"这么简单，而是四个视角产生**真实的矛盾和张力**。

### 四大师分工

| 大师 | 视角 | 核心问题 | 评分维度 |
|------|------|----------|----------|
| **段永平** | 商业模式本质 | "这门生意的本质是什么？" | 生意质量、商业模式 |
| **巴菲特** | 护城河+估值 | "护城河深不深？价格便宜吗？" | 财务健康、安全边际 |
| **芒格** | 逆向思考 | "反过来想，怎么会死？" | 风险清单、偏误自查 |
| **李录** | 长期确定性 | "10年后还在吗？确定性多高？" | 文明趋势、范式转移 |

### 对抗机制

**不是四份报告的拼接，而是四种思维方式的碰撞**：

- 段永平说"好生意"，芒格会问"怎么会死"
- 巴菲特说"够便宜"，李录会问"10年后还在吗"
- 如果四位大师意见一致 → 高确定性机会
- 如果大师之间有冲突 → 这才是投资决策的真实状态，需要权衡

### 四大师评分表

在分析结束时，必须输出四大师评分：

```
四大师评分：
- 段永平（商业模式）：X.X/5 — 核心判断
- 巴菲特（财务估值）：X.X/5 — 核心判断
- 芒格（逆向思考）：X.X/5 — 核心判断
- 李录（长期确定性）：X.X/5 — 核心判断

综合评分：X.X/5
```

**评分标准**：
- 4.5-5.0：极佳，强烈推荐
- 4.0-4.4：优秀，推荐
- 3.5-3.9：良好，有条件推荐
- 3.0-3.4：一般，灰色地带
- <3.0：不推荐

## 一、行业格局分析（自上而下）

**核心问题**：这个行业有没有技术壁垒？国产替代到哪一步了？当前处于周期哪个位置？

详细框架见 `industry-analyzer`。这里只做快速判断：

### 1.1 产业链定位

- 这家公司在上中下游哪个位置？
- 核心原材料是什么？供给是否受限？
- 终端客户是谁？需求是否可持续？

### 1.2 技术壁垒评估

| 壁垒等级 | 特征 | 典型行业 |
|---------|------|---------|
| 极高 | 技术密集、资金密集、认证周期长 | 半导体设备、IC载板 |
| 高 | 需要长期积累、客户粘性强 | 高端覆铜板、精密零部件 |
| 中 | 有一定技术门槛、可追赶 | PCB、锂电池材料 |
| 低 | 同质化竞争、拼成本 | 普通钢材、基础化工 |

### 1.3 行业周期位置

| 周期位置 | 特征 | 策略 |
|---------|------|------|
| 底部 | 产能过剩、价格战、厂商亏损 | 布局龙头，等待出清 |
| 上行 | 需求回暖、涨价、产能紧张 | 重仓持有 |
| 顶部 | 全行业暴利、疯狂扩产 | 逐步减仓 |
| 下行 | 需求下滑、库存累积、价格下跌 | 回避或做空 |

## 二、公司定位分析（自下而上）

**核心问题**：这家公司在行业里排第几？靠什么赚钱？对上下游有没有议价权？

### 2.1 主营业务构成

| 分析要点 | 具体内容 |
|---------|---------|
| 各业务营收占比 | 哪个是主力、哪个是增长点 |
| 各业务毛利率 | 哪个赚钱、哪个不赚钱 |
| 业务相关性 | 是否协同、还是杂乱拼凑 |

**关键判断**：
- 主营业务要聚焦，太杂的公司往往不专业
- 毛利率低于5%的业务，基本是给上游打工
- 高毛利率业务占比提升，说明结构优化

### 2.2 市场地位

| 地位 | 特征 | 投资价值 |
|-----|------|---------|
| 全球第一 | 规模或技术领先 | 最高 |
| 国内第一 | 国产替代龙头 | 高 |
| 细分龙头 | 某一细分领域领先 | 中高 |
| 跟随者 | 规模偏小、技术跟随 | 中低 |

### 2.3 核心客户

| 维度 | 看什么 | 预警信号 |
|------|--------|----------|
| 客户集中度 | 前五大客户占比 | >50%依赖大客户 |
| 客户质量 | 是否进入头部供应链 | 英伟达、苹果认证=高壁垒 |
| 客户稳定性 | 合作年限、复购率 | 频繁更换客户=产品力不足 |

### 2.4 上下游议价能力

| 维度 | 强势信号 |
|------|----------|
| 对上游 | 供应商分散、原材料占比低 |
| 对下游 | 客户分散、先收钱后发货 |
| 应收账款 | 周转快、占比低 |
| 存货 | 周转快=产品好卖 |

**关键判断**：
- 预收账款高 = 产品抢手，客户先付钱
- 应收账款高 = 被客户占用资金，议价权弱
- 存货周转天数暴增 = 卖不动了

### 2.5 国际化程度

| 维度 | 看什么 | 意义 |
|------|--------|------|
| 海外收入占比 | 国际化程度 | >30%说明有国际竞争力 |
| 海外产能布局 | 是否有海外工厂 | 规避关税、贴近客户 |
| 国际认证 | 是否进入国际供应链 | 英伟达、苹果认证=高壁垒 |
| 地缘政治风险 | 制裁/关税风险 | 半导体、军工敏感 |

## 三、管理层分析

**核心问题**：管理层是否可信？利益是否绑定？过往承诺是否兑现？

### 3.1 管理层背景

| 维度 | 好的信号 |
|------|----------|
| 专业背景 | 技术出身更踏实 |
| 行业经验 | 深耕多年 |
| 学历/职称 | 工程院院士、博士等 |

### 3.2 利益绑定

| 维度 | 好的信号 |
|------|----------|
| 持股比例 | >10%利益绑定 |
| 薪酬水平 | 合理水平 |
| 股权激励 | 有激励且目标明确 |
| 增减持 | 增持=看好，减持=套现 |

### 3.3 过往承诺兑现

| 维度 | 预警信号 |
|------|----------|
| 业绩承诺 | 连续不达标 |
| 募投项目 | 频繁延期、变更 |
| 分红承诺 | 说好的不分红 |
| 战略规划 | 画饼不兑现 |

### 3.4 资本运作记录

| 行为 | 好的信号 | 坏的信号 |
|------|----------|----------|
| 定增 | 用于主业扩产 | 频繁定增、用途不明 |
| 并购 | 整合能力强、标的优质 | 高溢价并购、商誉大 |
| 减持 | 少量减持、有合理解释 | 大额减持、清仓式减持 |
| 质押 | 低质押率 | 高质押率（>80%） |

## 四、财务数据验证

**核心问题**：赚钱能力怎么样？增长是否可持续？财务是否健康？

### 4.1 盈利能力

| 指标 | 公式 | 参考值 | 意义 |
|------|------|--------|------|
| 毛利率 | (营收-成本)/营收 | 制造业>20% | 产品竞争力 |
| 净利率 | 净利润/营收 | >10% | 综合盈利能力 |
| ROE | 净利润/净资产 | >15% | 股东回报效率 |
| ROIC | EBIT/(有息负债+净资产) | >10% | 资本回报效率 |

### 4.2 PEG指标（成长股估值利器）

**核心问题**：PE看着高，但增速能不能消化？

| 指标 | 公式 | 意义 |
|------|------|------|
| PEG | PE / 净利润增速(%) | 增速调整后的估值 |

**判断标准**：

| PEG | 判断 | 说明 |
|-----|------|------|
| <0.5 | 严重低估 | 增速远超估值，罕见机会 |
| 0.5-1 | 合理偏低 | 增速能消化估值，成长股甜蜜区 |
| 1-1.5 | 合理 | 估值与增速匹配 |
| 1.5-2 | 偏高 | 需要超预期增速才能撑住 |
| >2 | 高估 | 增速跟不上估值，危险 |

**使用场景**：
- 适用于**高增速成长股**（净利润增速>20%）
- 不适用于周期股（增速波动大，PEG无意义）
- 不适用于亏损公司（没有正增速）
- **和PE互补**：PE看绝对贵不贵，PEG看相对值不值

**案例**：
- 天赐材料PE=50倍，Q1净利+80%，PEG=50/80=0.63 → 看着贵其实不贵
- 某公司PE=30倍，Q1净利+15%，PEG=30/15=2.0 → 看着便宜其实贵了

### 4.3 个股PE与行业PE对比

**核心问题**：当前估值是贵还是便宜？和同行比有没有溢价？

**计算公式**：
```
溢价率 = (个股PE - 行业平均PE) / 行业平均PE × 100%
```

| 溢价率 | 判断 | 策略 |
|--------|------|------|
| <0% | 低估 | 可能有机会（但要查原因） |
| 0-30% | 合理 | 正常估值 |
| 30-80% | 偏高 | 需要增速或壁垒支撑 |
| 80-150% | 高估 | 谨慎，等待回调 |
| >150% | 严重高估 | 回避，风险极大 |

**判断逻辑**：
- **高增速可以消化高溢价**：如Q1净利+100%，PE高于行业平均50%仍可接受
- **高壁垒可以支撑高溢价**：如英伟达独家供应商，PE高于行业平均80%仍合理
- **低增速+高溢价=危险**：如Q1仅+14%，PE高于行业平均100%则严重高估
- **纯题材+高溢价=炒作**：如算力经销业务，PE高于行业平均300%则是纯炒作

**行业平均PE参考值（2026年）**：

| 行业 | 平均PE范围 | 说明 |
|------|-----------|------|
| 电子代工 | 20-25倍 | 毛利率低，估值偏低 |
| PCB | 30-40倍 | 技术壁垒中等 |
| 覆铜板 | 25-35倍 | 周期+成长双重属性 |
| 锂电材料 | 20-30倍 | 周期波动大 |
| 清洁能源 | 15-25倍 | 增速稳定但不高 |
| 半导体设备 | 40-60倍 | 高壁垒+高成长 |
| AI算力 | 30-50倍 | 高成长+高预期 |
| 消费电子 | 15-20倍 | 成熟行业 |

### 4.4 成长能力

| 指标 | 看什么 | 好的信号 |
|------|--------|----------|
| 营收增速 | 行业景气度 | >15% |
| 净利润增速 | 盈利弹性 | >营收增速（规模效应） |
| 扣非净利润增速 | 主业真实增长 | 与净利润增速匹配 |
| 经营现金流增速 | 赚的是真金白银 | 与净利润增速匹配 |

### 4.5 财务健康度

| 指标 | 参考值 | 预警信号 |
|------|--------|----------|
| 资产负债率 | <60% | >70%杠杆偏高 |
| 流动比率 | >1.5 | <1短期偿债困难 |
| 经营现金流 | 正 | 负=赚的是纸面利润 |
| 商誉/净资产 | <30% | >50%减值风险大 |
| 质押率 | <50% | >80%风险大 |

### 4.6 研发投入

| 指标 | 参考值 | 意义 |
|------|--------|------|
| 研发费用率 | 制造业>5% | 重视研发 |
| 研发人员占比 | >15% | 技术驱动 |
| 发明专利数量 | 行业前列 | 技术积累 |
| 研发资本化率 | <30% | 过高可能是美化利润 |

### 4.7 关联交易与治理风险

| 风险点 | 预警信号 |
|--------|----------|
| 关联交易 | 占营收>20% |
| 商誉 | 占净资产>30% |
| 质押率 | >80% |
| 对外担保 | 占净资产>50% |
| 未决诉讼 | 大额诉讼 |

## 五、股东博弈分析

**核心问题**：筹码在谁手里？大股东在想什么？

### 5.1 股东户数变化

| 变化 | 含义 | 策略 |
|------|------|------|
| 户数持续减少 | 筹码集中 | 可能有机会 |
| 户数暴增 | 散户蜂拥而入 | 危险信号 |
| 户数暴减 | 可能有大资金进场 | 关注 |

### 5.2 大股东动向

| 行为 | 信号 | 严重程度 |
|------|------|----------|
| 大股东增持 | 看好 | 正面 |
| 大股东减持 | 套现 | 负面 |
| 实控人变更 | 可能有新故事 | 中性偏正 |
| 股份被冻结/拍卖 | 公司治理风险 | 严重负面 |

### 5.3 机构持仓

| 维度 | 看什么 | 意义 |
|------|--------|------|
| ETF持仓 | 是否有主题ETF扎堆 | 赛道代表性 |
| 公募持仓 | 基金持股比例 | 机构认可度 |
| 北向持仓 | 外资持股比例 | 国际认可度 |
| 社保/养老金 | 长期资金持仓 | 安全性认可 |

## 六、产业链验证

**核心问题**：公司说的好，能不能从上下游数据验证？

### 6.1 验证方法

| 验证方式 | 具体做法 |
|---------|--------|
| 上游验证 | 看原材料供应商的出货量、价格走势 |
| 下游验证 | 看终端产品的销售数据、招标量 |
| 同行验证 | 看竞品的财报和表态 |
| 第三方数据 | 行业协会、研究机构数据 |

### 6.2 案例

- 验证PCB厂产能利用率：看覆铜板、铜箔出货量
- 验证风电叶片厂：看整机厂招标量
- 验证锂电池厂：看碳酸锂、正极材料出货量

## 八、Bull/Bear Case推演

**核心问题**：涨的话能涨多少？跌的话能跌多少？关键变量是什么？

### 8.1 Bull Case（看多情景）

- **触发条件**：什么情况下会涨？（行业需求超预期、国产替代加速、新产品放量、业绩超预期）
- **目标估值**：PE XX倍 × 预期EPS = 目标价 XX元
- **潜在涨幅**：XX%
- **概率判断**：高/中/低

### 8.2 Bear Case（看空情景）

- **触发条件**：什么情况下会跌？（行业周期下行、竞争加剧、大客户流失、政策利空）
- **目标估值**：PE XX倍 × 保守EPS = 目标价 XX元
- **潜在跌幅**：XX%
- **概率判断**：高/中/低

### 8.3 关键变量

- 最影响结果的那个变量是什么？价格？量？政策？
- 这个变量当前的趋势是什么？
- 什么信号出现意味着需要修正判断？

**使用要点**：
- Bull/Bear不是50:50，要根据当前信息给概率
- 概率会随着新信息（财报、政策、行业数据）动态调整
- 关键变量要具体到可追踪的指标

## 九、镜子测试（最终检验）

**核心理念**：如果5句话说不清楚为什么要买，那就不要买。没有例外。

### 镜子测试模板

在做出最终结论前，必须完成以下测试：

```
我以 [价格] 买入 [公司名称]，因为：

1. 这门生意的本质是 [一句话定义]，我理解它；
2. 它的护城河是 [核心竞争优势]，而且在 [变宽/变窄/稳定]；
3. 管理层 [评价]，值得 [信任/警惕]；
4. 当前价格相当于内在价值的 [X折]，[有/没有] 安全边际；
5. 即使我错了，下行风险可控，因为 [安全网]。
```

**判断标准**：
- ✅ 通过：5句话完整、逻辑自洽、每句都有具体依据
- ❌ 不通过：说不完整、逻辑矛盾、空洞无物

**不通过的情况**：
- "因为大家都看好" → 不买
- "因为最近涨了很多" → 不买
- "因为AI/新能源是趋势" → 不买（太笼统）
- "因为估值便宜" → 不买（没有说清楚为什么便宜）

**通过的情况**：
- "因为茅台的品牌护城河是时间积累的，无法复制，当前PE 25倍处于历史低位" → 可以买
- "因为腾讯12亿用户的社交关系链是垄断性的，账上净现金超2000亿提供安全网" → 可以买

## 附录：多Agent并行研究模式（借鉴AI Berkshire）

**核心理念**：4个独立Agent同时研究一家公司，每个Agent各自搜索、独立分析、独立给出评分，最后由主Agent综合研判。

### 适用场景

- 需要**最快速、最全面**的研究
- 信息量大，单Agent上下文窗口不够
- 想要**多视角对抗**，避免单一偏见

### 4个Agent分工

| Agent | 视角 | 职责 | 搜索关键词 |
|-------|------|------|-----------|
| **Agent 1** | 段永平（商业模式） | 分析生意本质、商业模式、竞争壁垒 | "{公司} 商业模式 护城河 竞争优势" |
| **Agent 2** | 巴菲特（财务估值） | 分析财务数据、估值水平、安全边际 | "{公司} 财报 估值 PE ROE" |
| **Agent 3** | 芒格（逆向思考） | 分析风险、竞争对手、失败情景 | "{公司} 风险 竞争对手 行业威胁" |
| **Agent 4** | 李录（长期确定性） | 分析管理层、行业趋势、10年前景 | "{公司} 管理层 行业趋势 长期前景" |

### 调用方式

使用 `delegate_task` 的 batch 模式，同时启动子agent。

**⚠️ 重要：`max_concurrent_children` 默认为3**，无法同时启动4个agent。解决方案是将芒格（风险）和李录（长期确定性）合并为一个agent：

```
delegate_task({
  tasks: [
    {
      goal: "分析{公司}的商业模式（段永平视角）",
      context: "你是投资分析师，以段永平的视角分析这家公司的商业模式。分析要点：1. 生意本质是什么？2. 商业模式是否可持续？3. 竞争壁垒在哪里？4. 对上下游有没有议价权？5. 评分(1-5分)。请搜索'{公司} 商业模式 护城河 竞争优势'等关键词获取最新信息，然后给出详细分析。",
      toolsets: ["web", "search"]
    },
    {
      goal: "分析{公司}的财务估值（巴菲特视角）",
      context: "你是投资分析师，以巴菲特的视角分析这家公司的财务和估值。分析要点：1. 财务健康度(ROE/负债率/现金流)？2. 估值水平(PE/PB/PEG)？3. 安全边际？4. 盈利能力趋势？5. 评分(1-5分)。请搜索'{公司} 财报 估值 PE ROE 2026'等关键词获取最新信息，然后给出详细分析。",
      toolsets: ["web", "search"]
    },
    {
      goal: "分析{公司}的风险与长期确定性（芒格+李录视角）",
      context: "你是投资分析师，以芒格的逆向思维和李录的长期视角分析这家公司。\n\n芒格视角分析要点：1. 反过来想，公司会怎么死？列出5大死亡情景及概率。2. 最大风险是什么？3. 竞争对手威胁？\n\n李录视角分析要点：4. 管理层是否可信？5. 10年后公司还在吗？更强还是更弱？6. 是否顺应文明趋势？7. 评分(1-5分)。请搜索'{公司} 风险 竞争 管理层 长期前景'等关键词获取最新信息，然后给出详细分析。",
      toolsets: ["web", "search"]
    }
  ]
})
```

**如果 `max_concurrent_children` 被调高到4+**，可以拆分为独立的4个agent以获得更深入的分析。

### 结果汇总

4个Agent返回后，主Agent（你）负责：

1. **汇总四大师评分**：
```
四大师评分：
- 段永平（商业模式）：X.X/5 — 核心判断
- 巴菲特（财务估值）：X.X/5 — 核心判断
- 芒格（逆向思考）：X.X/5 — 核心判断
- 李录（长期确定性）：X.X/5 — 核心判断
```

2. **识别冲突点**：
- 如果大师意见一致 → 高确定性
- 如果大师有分歧 → 列出分歧，这是决策的关键

3. **综合研判**：
- 加权平均（可根据情况调整权重）
- 最终结论

### 与单Agent模式的区别

| 维度 | 单Agent模式 | 多Agent并行模式 |
|------|------------|----------------|
| **速度** | 较慢（串行） | 快（并行） |
| **信息量** | 受上下文窗口限制 | 4倍搜索量 |
| **视角** | 可能有偏见 | 多视角对抗 |
| **深度** | 一般 | 更深（每个Agent专注一个维度） |
| **适用场景** | 日常分析 | 重要决策、深度研究 |

### 注意事项

1. **不要滥用**：多Agent模式消耗更多token，只在重要决策时使用
2. **子Agent无记忆**：每个子Agent是独立的，需要在context中提供完整信息
3. **结果需验证**：子Agent的分析可能有误差，主Agent需要交叉验证
4. **冲突是好事**：大师之间的分歧恰恰是投资决策的真实状态

### 使用示例

```
# 对协鑫能科进行多Agent并行研究
请用多Agent并行模式分析协鑫能科（002015）

# 系统会自动启动4个Agent：
# Agent 1 → 商业模式分析（段永平视角）
# Agent 2 → 财务估值分析（巴菲特视角）
# Agent 3 → 风险竞争分析（芒格视角）
# Agent 4 → 长期确定性分析（李录视角）

# 约2-3分钟后，汇总四大师评分和结论
```

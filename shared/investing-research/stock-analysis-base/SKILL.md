# 行业与个股分析 — 共享基础

本skill是分析框架的**底层基座**，提供所有分析skill共享的内容。做具体分析时，配合 `industry-analyzer`（行业）、`stock-analyzer`（个股）、`peer-comparator`（对比）使用。

---

## 〇、分析工作流（重要）

**当用户要求分析股票、行业、板块时，必须先加载相关skill：**

1. **个股分析** → 加载 `stock-analyzer` + `stock-analysis-base`
2. **行业分析** → 加载 `industry-analyzer` + `stock-analysis-base`
3. **公司对比** → 加载 `peer-comparator` + `stock-analysis-base`
4. **不确定** → 先加载 `stock-analysis-base`，再根据需要加载其他

**不要跳过skill加载直接用搜索硬做分析。** skill里有完整的分析框架、常见陷阱和参考案例，能保证分析质量一致。

### 实时数据获取模式

分析时通常需要实时数据，按以下顺序：
1. **实时行情**：`mcp_fetch_fetch` 抓取腾讯财经接口 `https://qt.gtimg.cn/q=sh{代码}` 或 `sz{代码}`
2. **新闻搜索**：`mcp_tavily_mcp_tavily_search` 搜索"{股票名} {日期} 新闻"
3. **深度信息**：`mcp_firecrawl_firecrawl_scrape` 抓取东方财富、雪球等页面
4. **财务数据**：使用 `a-share-data` 或 `tushare` skill

## 一、分析技巧

### 1. 先看风险，再看机会

对烂的东西毫不留情——基本面烂透了就直接说，别给人画饼。

### 2. 用数据说话，别用感觉

"光通信占比3.73%"——这就是数据，不是感觉。

### 3. 关注矛盾点

业绩巨亏，股价却能连板——找出矛盾，才能理解市场在炒什么。

### 4. 股东户数是散户情绪温度计

- 户数暴增 = 散户蜂拥而入 = 危险
- 户数持续减少 = 筹码集中 = 可能有机会

### 5. 毛利率是试金石

- <5%：给上游打工
- 5-15%：普通制造业
- 15-30%：有一定壁垒
- 30-50%：强技术或品牌溢价
- >50%：垄断或平台型公司

### 6. 产业链验证

公司说自己好不算好，要从上下游数据交叉验证。

### 7. 对比公司时必须明确业务范围

**Pitfall（用户纠正过）**：用户会问"你对比的是电缆业务还是光纤业务？"——当对比多家公司时，必须先明确对比的是哪个业务板块。

**做法**：
- 先列出各家公司的业务结构对比表
- 明确标注"本次对比主要围绕XX业务"
- 如果业务重叠度低，要说明哪些公司在这个业务上直接竞争、哪些不竞争

### 8. PEG是成长股的照妖镜

PE看着高不一定贵，PE看着低不一定便宜。PEG = PE / 净利润增速(%)，<1就是增速能消化估值。

### 9. 分析板块时要搜"跨界玩家"（用户纠正过）

**Pitfall（2026-06-23）**：用户问"绿电运营商头部公司"，我只列了华润、龙源、三峡等纯绿电运营商，漏掉了协鑫能科。但协鑫能科有"电力+算力"双轮驱动业务，是算电协同概念最纯正的标的之一。

**做法**：
- 分析某板块时，除了"纯血"公司，还要搜索"{板块名}+{相关赛道}"的交叉公司
- 例如：分析"绿电运营商"时，还要搜"绿电+算力"、"绿电+算电协同"
- 用 `mcp_tavily_mcp_tavily_search` 搜索"{板块} 概念股 龙头"可以发现更多标的

### 10. 区分"直接消耗"与"间接消耗"（用户纠正过）

**Pitfall（2026-06-23）**：分析算力金属板块时，用户问为什么没有推荐盛和资源（稀土龙头）。原因：稀土主要用于永磁电机（伺服电机），与AI是**间接关系**；而锡、钽、铟等是AI服务器/芯片的**直接消耗品**。

**做法**：
- 分析"XX金属/材料"板块时，必须区分该材料与目标应用场景的**绑定深度**
- 直接消耗（每台设备都要用）vs 间接消耗（通过中间环节关联）
- 在推荐时明确标注绑定度等级（⭐1-5星）

## 二、常见陷阱

| 陷阱 | 特征 | 如何避免 |
|------|------|----------|
| 壳股炒作 | 营收小、利润低、主业空心化 | 看主营毛利率 |
| 概念炒作 | 业务占比极小却被当主业炒 | 看各业务营收占比 |
| **市场预期与公司实际不符** | 券商研报/市场文章称公司产品已通过XX认证、签订XX长协，但公司官方公告明确否认 | **以公司官方公告为准**，如金钼股份2026-06-15公告"尚无产品直接应用于半导体存储芯片" |
| 财务洗澡 | 一次性大额计提 | 看扣非净利润 |
| 关联交易 | 与关联方大额交易 | 看关联交易占比 |
| 商誉减值 | 高溢价并购 | 看商誉/净资产比 |
| 大股东套现 | 减持+质押+冻结 | 看股东状态 |
| 跨界概念遗漏 | 分析某板块时只看"纯血"公司，遗漏跨界玩家 | 分析板块时，除了纯正标的，还要搜索"XX概念+其他赛道关键词"的交叉公司 |
| 电子级产品纯度陷阱 | 公司声称"高纯"不等于达到半导体级 | 4N（99.99%）≠ 5N（99.999%）≠ 6N（99.9999%），半导体级通常要求5N以上。必须验证公司公告，不能只看券商研报（金钼股份案例：市场炒作"以钼代钨"，公司明确否认半导体应用） |

## 三、飞书消息格式规范（重要）

**核心规则**：飞书文本消息不支持markdown表格渲染，会显示原始语法。**必须使用卡片消息+原生table组件展示表格数据，不要偷懒用纯文本。**

### 格式选择

| 场景 | 推荐格式 | 说明 |
|------|----------|------|
| 有表格数据 | 卡片消息（interactive）+ 原生table组件 | **首选方案**，支持表格渲染 |
| 简单列表 | 纯文本 + emoji + 分隔线 | 降级方案，用✓/✗做标记 |
| 需要加粗 | 富文本（post） | 支持加粗、链接等样式 |

**Pitfall（2026-06-22用户纠正）**：用户明确要求"尽可能不要使用md表格"。在普通对话中展示数据对比时，用纯文本列表、分条列示、emoji标记等方式，不要用markdown表格语法。飞书卡片+原生table组件仍然是首选。

### Lark-MCP调用稳定性（重要）

连续快速调用 `mcp_lark_mcp_im_v1_message_create` 会导致lark-mcp服务器崩溃。卡片消息内容尽量精简，如果连续调用失败，等待30-60秒服务器自动恢复。

